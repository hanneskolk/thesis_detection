# Military Object Detection System
> Bachelor's Thesis — Object detection application of military assets using YOLO11
> 
> Author: Hannes Kolk | University of Tartu, Institute of Computer Science

This repository contains the full pipeline for training, exporting, and deploying a military object detection system capable of identifying **7 target classes** in images and video. The system is built on [Ultralytics YOLO11](https://github.com/ultralytics/ultralytics) and served through a [Streamlit](https://streamlit.io) web interface with support for three inference backends.

## Requirements

**Python:** 3.10 or 3.11 recommended

**Hardware:**
- CPU-only: supported for inference (comment out `onnxruntime-gpu` and `tensorrt` in `requirements.txt`)
- NVIDIA GPU: required for TensorRT export and recommended for training
- Tested on: NVIDIA RTX A4000, CUDA 12.4

**Install all dependencies:**

```bash
pip install -r requirements.txt
```

The `requirements.txt` file includes `--extra-index-url` directives for `onnxruntime-gpu` and `tensorrt` so no additional install commands are needed under normal circumstances.

**If PyTorch installs without CUDA support** (verify with `python -c "import torch; print(torch.cuda.is_available())"`), override manually:

```bash
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124
```

**CPU-only machines:** edit `requirements.txt` before installing — comment out the `onnxruntime-gpu` and `tensorrt` blocks and replace with:

```
onnxruntime
```

## Quick Start

### Option A — Pre-trained model

If you have `best.pt` (and optionally `best.onnx`):

```bash
# 1. Clone and enter the repo
git clone <repo-url>
cd <repo-folder>

# 2. Create and activate a virtual environment
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Place model weights in the repo root
cp /path/to/best.pt .
cp /path/to/best.onnx .         # optional

# 5. Launch the app
streamlit run app.py --server.address 0.0.0.0 --server.port 8501
```

---

### Option B — Train from scratch, no prepared dataset

```bash
# 1. Clone and enter the repo
git clone <repo-url>
cd <repo-folder>

# 2. Create and activate a virtual environment
python3 -m venv venv
source venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Place raw images and YOLO labels into raw_dataset/
#    Accepted layouts:
#      raw_dataset/*.jpg  +  raw_dataset/*.txt
#      raw_dataset/images/*.jpg  +  raw_dataset/labels/*.txt

# 5. Prepare the dataset
python dataset.py

# 6. Train
python main.py --data dataset/data.yaml

# 7. Copy best weights to the app root
cp results/detector/weights/best.pt .

# 8. Launch the app
streamlit run app.py --server.address 0.0.0.0 --server.port 8501
```

## Dataset Preparation

`dataset.py` scans a raw source folder, pairs each image with its YOLO `.txt` label file, splits the data, and writes the prepared dataset with a `data.yaml` config.

**Default split:** 80% train / 15% valid / 5% test

```bash
# Default — reads raw_dataset/, writes dataset/
python dataset.py

# Custom source and output paths
python dataset.py --source /path/to/raw --output my_dataset

# Override class names (must match label file index order)
python dataset.py --classes Artillery Missile Radar RocketLauncher Soldier Tank Vehicle

# Custom split ratios
python dataset.py --train 0.85 --val 0.10
```

**Expected label format:** standard YOLO — one `.txt` per image, each line:
```
<class_id> <cx> <cy> <width> <height>
```
All values normalised to [0, 1] relative to image dimensions.

## Training

`main.py` runs the full training pipeline: optional dataset preparation, YOLO11 fine-tuning, validation, and ONNX export.

```bash
# Full pipeline — prep dataset then train
python main.py

# Skip dataset prep, use existing data.yaml or dataset.yml
python main.py --data dataset/dataset.yml

# Larger model (better accuracy, more VRAM required)
python main.py --data dataset/data.yaml --model yolo11m.pt

# Extended training run
python main.py --data dataset/data.yaml --epochs 150 --batch 32

# CPU-only machine
python main.py --data dataset/data.yaml --device cpu --batch 8

# Train and export ONNX automatically
python main.py --data dataset/data.yaml --export-onnx

# Give the run a distinct name to avoid overwriting previous results
python main.py --data dataset/data.yaml --name detector_v2
```

**Available base models:**

| Model | Size | Speed | Accuracy |
|---|---|---|---|
| `yolo11n.pt` | Nano | Fastest | Lowest |
| `yolo11s.pt` | Small | Fast | Good — **default** |
| `yolo11m.pt` | Medium | Moderate | Better |
| `yolo11l.pt` | Large | Slow | High |
| `yolo11x.pt` | Extra-large | Slowest | Highest |

Training output is written to `results/<run-name>/`. After training, promote the best weights:

```bash
cp results/detector/weights/best.pt .
```

## Inference Backends

The app supports three backends selectable from the sidebar, with automatic fallback:
**TensorRT → ONNX → PyTorch**

| Backend | File | Speedup | Requirements |
|---|---|---|---|
| PyTorch | `best.pt` | 1× (baseline) | None beyond base install |
| ONNX | `best.onnx` | ~1.6× | `onnxruntime-gpu` |
| TensorRT | `best.engine` | ~3.4× | NVIDIA GPU, CUDA, TensorRT |

**Benchmarked on RTX A4000 (n=100 images, 640×640):**

| Backend | Mean (ms) | Std (ms) | P95 (ms) | FPS |
|---|---|---|---|---|
| PyTorch (.pt) | 19.85 | 17.75 | 58.89 | 50.4 |
| ONNX (.onnx) | 12.36 | 1.57 | 14.93 | 80.9 |
| TensorRT (.engine) | 5.77 | 0.55 | 6.39 | 173.4 |

**ONNX export** — via the app sidebar "Export ONNX" button, or:
```bash
python main.py --data dataset/data.yaml --export-onnx
cp results/detector/weights/best.onnx .
```

**TensorRT export** — via the app sidebar "Export TRT" button. The `.engine` file is compiled for the specific GPU it was built on and cannot be transferred to a different GPU model.

## Running the App

### Local

```bash
streamlit run app.py
```

Opens at `http://localhost:8501`.

### Remote VM / TensorDock

```bash
streamlit run app.py --server.address 0.0.0.0 --server.port 8501
```

**SSH tunnel** (recommended — no port needs to be publicly exposed):
```bash
# Run on your local machine
ssh -L 8501:localhost:8501 user@<vm-ip> -p <ssh-port>
# Then open http://localhost:8501 in your local browser
```

**Exposed port** (via TensorDock dashboard → Ports → expose 8501):
```
http://<vm-ip>:<mapped-port>
```

**Persistent config** — create `.streamlit/config.toml` in the repo root:
```toml
[server]
port = 8501
address = "0.0.0.0"
headless = true
```

## App Features

### Image Detection
Upload a JPG or PNG. The model runs inference and displays the original and annotated images side by side. Each detection is listed with its class name, confidence score, and bounding box coordinates. A stats overlay shows object count, mean confidence, inference time, and active backend.

### Video Detection
Upload MP4, AVI, MOV, or MKV. Processes frame by frame with optional **ByteTrack object tracking** — each detected object receives a persistent ID across frames and its trajectory is drawn as a trail. The processed video is available for in-browser playback and download.

### Validation
Point the app at your dataset root folder. The app samples images from the selected split, runs inference, compares against YOLO ground truth labels, and reports:

- Precision, Recall, F1, and mean IoU
- Per-class object count vs prediction count table
- Confidence score distribution histogram
- Annotated sample image grid (GT in green, predictions in blue)
- JSON export for cross-run comparison

## Benchmarking

`benchmark.py` measures per-image inference latency across all three backends and outputs a formatted table and CSV.

```bash
# Default — uses dataset/test/images, 100 images, 10 warmup frames
python benchmark.py

# Custom image directory and sample size
python benchmark.py --images dataset/test/images --n 200 --warmup 20

# Skip TensorRT if not available
python benchmark.py --no-trt

# Save results to a custom path
python benchmark.py --out my_benchmark.csv
```

## Model Versioning

Use distinct `--name` values for each training run to prevent overwriting:

```bash
python main.py --data dataset/data.yaml --name detector_v1
python main.py --data dataset/data.yaml --name detector_v2 --model yolo11m.pt
```

Compare runs using the Validation tab — load each `best.pt` via the sidebar path input and run against the same split. Export the JSON results from each run to compare metrics.

To promote a new model to active use:

```bash
cp results/detector_v2/weights/best.pt .
cp results/detector_v2/weights/best.onnx .   # if exported
```

The TensorRT engine must always be rebuilt after replacing `best.pt` — use the "Export TRT" button in the app sidebar.
